# Day 5 — Shell Scripting & Automation (DevOps)

Notes for Day 5 of the [5-Day Linux for DevOps Plan](../Linux-5-Day-Learning-Plan.md).

This is where everything comes together: turn the commands from Days 1–4 into
**repeatable automation**.

## Topics
1. [Bash Scripting Basics](01-bash-scripting-basics.md) — shebang, variables, args, exit codes.
2. [Control Flow & Functions](02-control-flow-and-functions.md) — `if`, `case`, loops, functions.
3. [Cron & Automation](03-cron-and-automation.md) — `cron`, `systemd` timers.
4. [Logs & Troubleshooting](04-logs-and-troubleshooting.md) — debugging methodology.
5. [Linux on GCP](05-linux-on-gcp.md) — startup scripts, `gcloud`, tying it together.

## Day 5 Goals
- [ ] Write and run a Bash script with variables, arguments, and exit codes.
- [ ] Use conditionals, loops, and functions to add logic.
- [ ] Schedule a job with `cron`.
- [ ] Debug a script (`set -x`, `set -e`) and read system logs.
- [ ] Bootstrap a **GCP VM** with a startup script — full circle to the GCP plan.
