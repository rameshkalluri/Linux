# Day 4 — Networking & Remote Access

Notes for Day 4 of the [5-Day Linux for DevOps Plan](../Linux-5-Day-Learning-Plan.md).

## Topics
1. [Networking Basics](01-networking-basics.md) — IP, ports, DNS, the OSI/TCP model.
2. [Network Commands](02-network-commands.md) — `ip`, `ping`, `curl`, `ss`, `dig`, `nc`…
3. [SSH & Remote Access](03-ssh-and-remote-access.md) — keys, `scp`, config, GCP VMs.
4. [Firewall & Security](04-firewall-and-security.md) — `ufw`, `iptables`, ports.

## Day 4 Goals
- [ ] Explain IP addresses, ports, and how DNS resolves a name.
- [ ] Diagnose connectivity with `ping`, `curl`, `ss`, `dig`, `traceroute`.
- [ ] SSH into a remote server using **key-based auth** and copy files with `scp`.
- [ ] Open/close ports with a firewall and verify what's listening.
- [ ] Run the "site is down" troubleshooting playbook end-to-end.
